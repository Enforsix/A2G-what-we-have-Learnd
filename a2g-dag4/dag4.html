Nettside analyse

Hei! I dag har vi vært gjennom litt hvordan vi skal tenke og hvordan vi kan analysere en nettside for å få forståelse og inspirasjon til hvordan nettsider er bygget opp før vi begynner å designe og kode dem selv. Selv om vi nå har sett på teorien rundt HTML og CSS, gir det utrolig mye verdi å analysere nettsider visuelt og i koden for å forstå både oppsett og virkemåte. Dette gir oss innsikt i hvordan ulike elementer brukes på tvers av nettsteder og hjelper oss å bygge en god, solid forståelse for oppbyggingen. Gjennom å bruke verktøyet kan vi dykke dypere inn i koden til nettsiden og få tilgang til detaljer som vi ikke nødvendigvis ser ved første øyekast. Her får vi oversikt over hvordan HTML-elementer er strukturert, hvilke CSS-egenskaper som er brukt, og hvordan de sammen skaper nettsidens endelige utseende. Dette lar oss forstå mer om blant annet oppsett av layouter, navigasjons menyer og responsivt design. Semantisk HTML er spesielt viktig for å oppnå et universelt og tilgjengelig design. Ved å bruke semantiske elementer som header, nav, main, footer, og så videre, blir nettsiden ikke bare mer forståelig for andre utviklere, men også for søkemotorer og hjelpe teknologi som skjermlesere. Samtidig gir CSS Flex oss fleksibilitet til å tilpasse layouten på tvers av skjermstørrelser, noe som er essensielt for dagens webdesign. Ved å fortsette å analysere og bryte ned eksisterende nettsider på denne måten får vi et godt grunnlag for å designe og kode våre egne nettsider i tråd med moderne praksis og standarder. Oppgave beskrivelse: Fortsett videre på arbeidet ditt med semantisk HTML, og bygg videre på nettsiden din ved å ta i bruk CSS Flex som plasserings metode. I denne oppgaven skal du fokusere på bruker reisen og analysere hvordan nettsidens struktur og layout kan forbedres for å gi en optimal opplevelse. Tenk på følgende spørsmål: Hvem: Hvem er målgruppen din? Hvordan kan layouten og strukturen tilpasse seg brukernes behov? Hva: Hva skal nettsiden formidle eller tilby? Sørg for at informasjonen er logisk organisert og enkelt tilgjengelig. Hvor: Hvor på siden bør ulike elementer plasseres for å være intuitive og lett tilgjengelige? Tips: Gjerne fortsett å lag en portfolio stil nettside som du kan bruke som din hjemmeside for prosjekter du lager.

Krav:

  • Fortsett å benytte deg av semantiske elementer, som header, main, section, nav, og footer for å bygge opp nettsiden på en strukturert og forståelig måte. Elementer som div og span kan brukes der det ikke er et semantisk alternativ
  • CSS Flex for layout på siden. Tenk på hvordan elementene skal plasseres og tilpasses forskjellige skjermstørrelser for å forbedre brukeropplevelsen.
  • Bruk analysemetoden vi har gått gjennom for å se på hvordan andre nettsteder er bygd opp. La deg inspirere av hvordan de har brukt flex og organisert innholdet for å treffe sine brukeres behov.

  • Legg til skygger for å lage dybde på nettsiden din. (Hint: Her må dere søke opp box-shadow, og eventuelt text-shadow ut i fra hvor dere vil ha skygger)
